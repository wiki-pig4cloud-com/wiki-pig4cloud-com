### 配置终端不校验验证码

在 `nacos` 的 `pig-auth-dev.yml` 配置文件中，可以通过以下配置来设置不校验验证码的终端：

```yaml
security:
  ignore-clients:
    - test
    - XXX
```

将需要忽略验证码校验的客户端 ID 添加到 `ignore-clients` 列表中即可。

### 配置前端关闭验证码

在 `env.ts` 文件中，可以通过配置 `VITE_VERIFY_ENABLE` 来控制是否显示验证码：

```typescript
VITE_VERIFY_ENABLE = false // 是否开启验证码校验
```

- 当 `VITE_VERIFY_ENABLE = true` 时，系统会显示验证码
- 当 `VITE_VERIFY_ENABLE = false` 时，系统将不显示验证码

<Note>
修改配置后需要重启服务才能生效。
</Note>
