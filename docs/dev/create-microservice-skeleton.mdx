## 1. 编译原有 pig 项目

在 pig 根目录执行以下命令：

```bash
mvn clean install
```

## 2. 执行生成

<Warning>
**务必按照本文档** `包名、项目名` **进行初始化，其他请了解 pig** `feign、mybatis` **的包扫描规则，不然 bean 扫描不到！**

## 常见问题

### 1. 运行提示缺少依赖

<Warning>
运行新模块提示缺少依赖报错，如下图报错
</Warning>

![缺少依赖报错](/images/1661477533668-37521260-69b3-4fa5-ab46-02501843a0a9.png)

### 2. 解决方案

<Tabs>
  <Tab title="检查编译">
    **编译验证：** 确认已完成PIG微服务项目的整体编译（执行过`mvn clean install`）
  </Tab>
  <Tab title="检查版本">
    **版本一致性：** 核对新建项目与PIG主工程的版本号（查看pom.xml中parent版本号）
  </Tab>
  <Tab title="检查 Maven 设置">
    **配置验证：** 验证IDEA中Maven配置指向正确的本地仓库及远程仓库（File > Settings > Build Tools > Maven）
  </Tab>
</Tabs>
