在进行新项目的技术选型时，强烈推荐使用 **master 分支（默认）** 的技术栈：Java17 + SpringBoot3.5 + Spring Cloud 2025 + Vue3。

<Note>
master 分支是当前的主要开发分支，提供最新的技术特性和最佳的性能表现。
</Note>

## 技术栈优势

<CardGroup cols={3}>
  <Card title="master" icon="java">
    长期支持版本，相较于 Java8 带来显著的性能提升和新特性支持
  </Card>
  <Card title="SpringBoot 3.5" icon="leaf">
    最新版本框架，提供更好的云原生支持和性能优化
  </Card>
  <Card title="Spring Cloud 2025" icon="cloud">
    微服务架构的最新实现，支持现代化的分布式系统开发
  </Card>
</CardGroup>

## 分支版本说明

<AccordionGroup>
  <Accordion title="master 分支（推荐使用）" icon="star">
    **核心技术栈**：Java17 + SpringBoot 3.5 + Spring Cloud 2025 + Vue3
    
    <Tip>
    这是当前的默认分支，推荐用于所有新项目开发。
    </Tip>

    **技术优势**：
    - 🚀 **性能优化**：Java17 带来显著的性能提升和内存优化
    - 🛡️ **安全性**：最新的安全特性和漏洞修复
    - 🔧 **开发效率**：SpringBoot 3.5 提供更丰富的开发工具和自动配置
    - ☁️ **云原生**：Spring Cloud 2025 完美支持容器化和微服务架构

    <CardGroup>
      <Card title="前端仓库" icon="link" href="https://gitee.com/log4j/pig-ui">
        JDK17 版本的前端代码仓库
      </Card>

      <Card title="后端仓库" icon="link" href="https://gitee.com/log4j/pig/tree/next-sa">
        JDK17 版本的后端代码仓库
      </Card>
    </CardGroup>
  </Accordion>

  <Accordion title="java8_eof 分支（过时，后续迁移）" icon="archive">
    **核心技术栈**：Java8 + SpringBoot 2.7 + Spring Cloud 2023 + Vue3
    
    <Warning>
    此分支仅用于维护现有项目，不推荐用于新项目开发。
    </Warning>

    **注意事项**：
    - ⚠️ **技术栈过时**：使用的技术版本相对较旧
    - 🔒 **安全风险**：可能存在已知的安全漏洞
    - 📉 **性能限制**：无法享受新版本带来的性能提升
    - 🚫 **功能受限**：缺少最新的框架特性和功能
    - 🔄 **迁移计划**：后续将升级到 JDK17 技术栈
  </Accordion>
</AccordionGroup>

## 迁移建议

<Steps>
  <Step title="评估现有项目">
    分析当前项目的技术债务和迁移成本
  </Step>
  <Step title="制定迁移计划">
    规划从 Master 分支到 JDK17 分支的迁移路径
  </Step>
  <Step title="逐步迁移">
    建议采用渐进式迁移策略，降低风险
  </Step>
  <Step title="测试验证">
    完整的功能测试和性能测试验证
  </Step>
</Steps>
